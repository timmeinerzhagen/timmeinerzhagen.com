Introduction

This personal website was created as a byproduct of other projects. A domain was needed to make the endpoints of a separate project reachable through the internet. After these endpoints were setup, a domain was available - rather than wasting web real estate - a personal website was created. It started just as a placeholder inclding name and some socials links, but in the end it was extended other time.

All in all it is a project, that was created as a learning experience allowing the application of practices in web development, website hosting and management, later Cloud hosting, Search Engine Optimization, Content Distribution Networks, legal requirements and most of all patience.

Infrastrucure

Initially the website was hosted on a personal Raspberry Pi running Nginx in a Docker container. After some network issues, problems with uptime, and long request delays a different approach was needed.

Firstly, a lift-and-shift approach was taken, simply running the existing Nginx Docker image on a Google Compute Cloud Comute Engine. The advantage of Compute Engine was that a small machine would be well withing the free tier of Google Cloud Platform. This allowed for a completely free website hosting, but with the drawback of a relatively weak machine, having only 500MB RAM and one virtual compute core. THis in conjunction of having to use docker led to much overhead on limited resources.

The final nail in the coffing for the Compute Engine approach was, that Google removed static IPv4-addresses from their free tier. This destroyed the "free website" approach at first. A workaround was quickly found using DynamicDNS, which simply changes the DNS entry every time Google assigns a new dynamic IPv4 address is assigned to the Compute Engine. This still led to short outages during the times an IP change stays undetected or when the DNS entry changes need to be established and accepted. In the end a lasting change needed to be done.

Today, the website is still hosted on Google Cloud Platform but now using Google App Engine, a service offering fully managed frontend and backends for website. This allowed to discard any self hosting components including Nginx and Docker, only leaving the website itself to worry about. Even the "free website" label was available again since Google App Engine offer 28 hours of frontend-instances for free every day. This even allows dynamically scaling to more than one frontend instance in case of usage spikes. In the end this cloud native approach allowed for a reduction in management needed for the side, while also being pretty much free.

Content and Frontend

The acual website is written in plain HTML, CSS and JavaScript. The intention of this is to minimize dependencies on bloated 3rd party imports. Initially, this path was taken to improve PageInsights scores in order to impact SEO ratings, but in the end it just enabled more in deph approach into how webpages function and how their performance is scored.

The first draft of the website was based on an existing template called "Halcyon Days" provided by Codrops. Many parts of this template are still in use, such as the background image on the main page and CSS/HTML elements. After some time, most unused parts of the template like additional pages and sections were removed. In a next step, entire files and libraries were inlined into custom scripts, in order to minimize the footprints of included files.

All these efforts led the page to get a total PageInsights score of 100 for Desktop and around 97 for Mobile. These differences are because of differing draw times for these systems. Desktops generally faster times for every page. Mobile pages need further special improvements, which would probably need a dedicated mobile version - a measure that is probably overkill for this project, but might be a good exercise for the future.
